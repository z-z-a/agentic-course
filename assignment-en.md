# Assignment: Job Search Tracker

## Background

Job searching is a repetitive, multi-stage process. You send CVs, receive responses (or silence), schedule interviews, get offers or rejections — and somehow need to keep track of it all. Email is where most of this communication happens, but it's scattered, unstructured, and easy to lose track of.

## Goal

Build a **Job Search Dashboard** — a personal tool that helps a job seeker track and understand the status of every job application they've submitted.

The dashboard must:

1. **Allow the user to manually log job applications** — including details such as the company name, role, a link to the job description, and the CV file that was sent.

2. **Connect to the user's email account** and identify emails related to job applications (sent CVs, recruiter replies, interview invitations, rejections, offers, etc.)

3. **Analyze recruiter emails** to determine the current status of each application — e.g., whether the recruiter acknowledged receipt, requested an interview, rejected the candidate, or went silent.

4. **Allow manual editing of application status and notes** — the user should be able to override or adjust the automatically detected status, and add free-text notes to any application.

5. **Present a clear, organized view** of all job applications and their current status, so the user can see at a glance where they stand with every company they've applied to.

## What you should demonstrate

- Working with an external service (email) programmatically
- Using AI to extract meaning from unstructured text (recruiter emails)
- Building a functional UI that surfaces actionable information
- Applying agentic development practices learned in the course (using AI tools to accelerate design, implementation, and debugging)

## Deliverable

A working application with a connected GitHub repository.
