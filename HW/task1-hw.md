# תרגיל 1: מימוש בסיסי

## סקירה כללית

זהו התרגיל הראשון בקורס  לפיתוח אגנטי. תרגיל 1 נועד לבסס את קו הבסיס האישי שלכם בפיתוח בסיוע AI **לפני** שתלמדו טכניקות שיטתיות.

תממשו יכולות סינון מוצרים גם ב-API של הצד השרת (backend) וגם בממשק הצד הלקוח (frontend) של קטלוג מוצרי מסחר אלקטרוני בסיסי מאוד (נמצא ב-`app/`), בדיוק כפי שהייתם עובדים בפרויקט אמיתי היום.

**חשוב**: זהו מדד בסיס, לא מבחן. השתמשו בעוזרי קוד AI בצורה שהכי נוחה לכם. אין כללים, הגבלות או דרישות לגבי אופן השימוש בכלי AI - פשוט עבדו בצורה טבעית.

## התחלה מהירה

הריצו את האפליקציה בשני טרמינלים כדי שתוכלו לבדוק אותה לפני התרגיל:

**טרמינל 1 - צד שרת**:
```bash
cd app/backend
uv venv --python 3.12
uv sync
uv run python run_api.py
# → http://localhost:8000
```

**טרמינל 2 - צד לקוח**:
```bash
cd app/frontend
npm install -g bun
bun install
bun dev
# → http://localhost:3000
```

## מטרות

1. **לממש פיצ'ר fullstack מלא** מצד השרת ועד צד הלקוח
2. **לבסס את קו הבסיס שלכם** מבחינת זמן, מאמץ וביטחון בשימוש בעוזרי AI
3. **לחוות את תהליך העבודה הנוכחי שלכם** לפני שתלמדו טכניקות אופטימיזציה לתרגילים הבאים

**הערה חשובה**: סביר שהמימוש שלכם יעבוד, גם עם הנחיות פשוטות - עוזרי קוד AI חזקים מספיק כדי לטפל בפיצ'רים האלה. הפרויקט הזה פשוט בכוונה כי **המטרה היא לא להתמודד עם קוד שלא עובד**. במקום זאת, שימו לב ל:
- כמה חלק התהליך מרגיש?
- כמה אתם מבינים מה ממומש?
- כמה שליטה אתם מרגישים שיש לכם על פרטי המימוש הספציפיים?
- האם אתם בכיסא הנהג, או סתם נוסעים?

## המשימה

> פתחו את עוזר הקוד AI שלכם בתיקיית `app/` בשביל זה!

המשימה מורכבת משני חלקים, אבל כמובן שאפשר לבקש לממש את שניהם בבת אחת אם רוצים:

### חלק 1: סינון ב-API של צד השרת

הוסיפו יכולות סינון ל-backend של FastAPI:

- **סינון לפי מחיר**: תמיכה בסינון מחיר מינימום ומקסימום
- **סינון לפי קטגוריה**: אפשרות לסנן לפי קטגוריית מוצר
- **חיפוש לפי מילת מפתח**: חיפוש בשמות ותיאורי מוצרים
- **מיון**: אפשרות מיון לפי מחיר או שם (בשני הכיוונים)

כל הסינונים צריכים להיות אופציונליים ולעבוד יחד כשהם משולבים.

### חלק 2: ממשק סינון בצד הלקוח

בנו ממשק React שמתחבר ל-API הסינון של צד השרת:

- **בקרות טווח מחירים**: אפשרו למשתמשים להגדיר סינון מחיר מינימום/מקסימום
- **בורר קטגוריות**: אפשרו סינון לפי קטגוריית מוצר
- **שדה חיפוש**: ספקו פונקציונליות חיפוש לפי מילת מפתח
- **בקרות מיון**: אפשרו למשתמשים למיין תוצאות
- **ניהול סינונים**: אפשרות להחיל ולנקות סינונים

הממשק צריך לטפל במצבי טעינה, תוצאות ריקות ושגיאות ולידציה בצורה מתאימה.

### השתמשו ב-AI איך שבא לכם

**אין כללים או הגבלות** על שימוש ב-AI:

- **עבדו בדיוק כפי שהייתם עובדים אם הייתם מקבלים את המשימה הזו בפרויקט אמיתי היום**
- השתמשו בכל סגנון הנחיות שמרגיש לכם טבעי
- בקשו עזרה, יצירת קוד, דיבוג - מה שצריך

## מה לעקוב אחריו

תוך כדי עבודה, עקבו אחר המדדים הבאים:

### מעקב זמן

- **זמן צד שרת**: כמה זמן לקח לממש את נקודת ה-API החדשה?
- **זמן צד לקוח**: כמה זמן לקח לבנות את רכיבי הממשק?
- **זמן כולל**: משך התרגיל הכולל

### אינטראקציה עם AI

- **מספר הנחיות**: כמה פעמים הנחיתם את עוזר ה-AI?
- **סוגי בקשות**: ביקשתם יצירת קוד? דיבוג? הסברים?
- **מחזורי איטרציה**: כמה חילופי הודעות נדרשו?

### רמת ביטחון

לאחר השלמת שתי המשימות, דרגו את הביטחון שלכם (1-10):

- כמה אתם בטוחים שהקוד **נכון**?
- כמה אתם בטוחים שהקוד עוקב אחר **שיטות עבודה מומלצות**?
- כמה אתם **מבינים** את הקוד שנוצר?
- כמה **תחזוקתי** הקוד שהתקבל?

### בעיות שנתקלתם בהן

- האם ה-AI עשה טעויות? מאיזה סוג?
- האם נדרשתם לדבג קוד שנוצר?
- האם היו שגיאות טיפוסים או כשלים בבדיקות?
- כמה תיקון ידני נדרש?

## קריטריונים להצלחה

סיימתם את התרגיל כאשר:

- כל בדיקות צד השרת עוברות (`uv run pytest`)
- ה-API של צד השרת מקבל ומעבד פרמטרי סינון בצורה נכונה
- צד הלקוח מציג ממשק סינון עובד
- ניתן להחיל סינונים והתוצאות מתעדכנות בהתאם
- ניתן לבדוק ידנית את תהליך הסינון המלא בדפדפן

## הערות חשובות

### זו לא תחרות

המטרה היא לבסס את **קו הבסיס האישי שלכם**, לא להתחרות באחרים. היו כנים לגבי:

- זמן שנדרש (אל תמהרו!)
- הנחיות ששימשו (עקבו אחרי כולן)
- רמות ביטחון (היו ריאליסטים)
- בעיות שנתקלתם בהן (תעדו הכל)

### תיעוד הוא המפתח

לאחר השלמת התרגיל, תחשבו על:

- מה עבד טוב בתהליך העבודה שלכם עם AI?
- מה היה מתסכל או איטי?
- איפה נתקעתם?
- מה הייתם רוצים לשפר?

המחשבות האלה ישמשו בסיס למודולי הלמידה שיבואו בהמשך.

## לאחר השלמה

כשתסיימו, יהיה לכם:

1. פיצ'ר סינון fullstack עובד
2. מדדי בסיס לתהליך העבודה שלכם בפיתוח בסיוע AI
3. מודעות לחוזקות ונקודות הכאב הנוכחיות שלכם
4. הקשר לטכניקות האופטימיזציה שתלמדו בהמשך
